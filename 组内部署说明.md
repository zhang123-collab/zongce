# 组内完整数据部署说明

本压缩包仅供项目组成员进行 Docker 部署与答辩测试。包内包含真实学生证明材料、当前 SQLite 数据库、上传文件、DeepSeek 配置和应用密钥，严禁上传公开仓库、公共网盘或无关群聊。

## 启动步骤

1. 通过可信方式取得压缩包并完整解压。
2. 启动 Docker Desktop，等待 Docker Engine 运行。
3. 如果本机8000端口已有项目运行，先停止原服务。
4. 双击 `Docker组内启动.bat`。
5. 访问 http://127.0.0.1:8000，API文档为 http://127.0.0.1:8000/docs。

也可以执行：

```powershell
docker compose -f docker-compose.yml -f docker-compose.internal.yml up --build -d
```

停止服务：

```powershell
docker compose -f docker-compose.yml -f docker-compose.internal.yml down
```

## 数据说明

- `internal-data/zongce.db`：打包时创建的一致性数据库副本，不是直接复制运行中的数据库文件。
- `internal-data/uploads`：系统上传目录。
- `internal-data/student-materials`：原学生证明材料。
- `internal-data/evidence`：数据库引用但不在上述目录内的其他材料。
- `.env`：当前DeepSeek等本机配置。
- `.zongce_secret`：当前应用签名密钥。

数据库副本中的材料路径已转换为容器内 `/app/internal-data/...` 路径。不要单独移动 `internal-data` 中的文件。

默认账号密码仍为 `123456`。组内联调结束后，应删除不再需要的压缩包和解压目录；如果压缩包离开可信范围，应立即轮换 DeepSeek API 密钥。
