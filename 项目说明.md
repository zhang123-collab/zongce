# 本科生综合测评自动计分系统——项目说明

## 1. 项目简介

本项目是基于 FastAPI、SQLAlchemy、SQLite 和原生 JavaScript 实现的本科生综合测评管理系统。系统围绕学生申报、证明材料、教师审核、成绩核算、终审、公示与异议建立完整业务闭环，并提供材料脱敏、DeepSeek 辅助核验、重复材料检测、排名、Excel 导出和个人 PDF 报告。

DeepSeek 只提供人工复核建议，不能自动修改规则分、审核状态或最终成绩；AI 服务不可用时不影响人工审核和核算主流程。

## 2. 技术架构

- 后端：Python 3.11、FastAPI、SQLAlchemy
- 数据库：SQLite
- 前端：HTML、CSS、原生 JavaScript
- 身份认证：JWT Bearer Token
- 文件处理：PDF/JPG/PNG、XLSX、可选本地 OCR
- AI 服务：DeepSeek Chat Completions API
- 容器部署：Docker、Docker Compose
- 默认端口：8000

主要目录：

```text
app.py                 FastAPI 应用组合与启动入口
zongce/                业务模型、服务和路由
static/                三角色前端页面
tests/                 自动化回归测试
scripts/               环境检查、备份与交付打包工具
deploy/demo_data.sql   Docker 首次启动使用的脱敏演示数据
```

## 3. Docker 本地部署（验收推荐）

### 3.1 前置条件

安装 Docker Desktop，并确认 Docker Desktop 已启动：

```powershell
docker --version
docker compose version
```

### 3.2 启动

Windows 用户可直接双击 `Docker启动.bat`，停止时双击 `Docker停止.bat`。

也可以在项目目录打开 PowerShell：

```powershell
docker compose up --build -d
docker compose ps
```

浏览器访问：

- 系统首页：http://127.0.0.1:8000
- API 文档：http://127.0.0.1:8000/docs

查看日志：

```powershell
docker compose logs -f zongce
```

停止服务但保留数据库和上传材料：

```powershell
docker compose down
```

重新启动：

```powershell
docker compose up -d
```

只有明确需要清空全部 Docker 演示数据时才执行：

```powershell
docker compose down -v
```

SQLite 数据库和上传材料保存在 Docker 命名卷 `zongce_data` 中，普通 `docker compose down` 不会删除数据。

## 4. Windows 非 Docker 部署

推荐依次双击：

1. `安装依赖.bat`
2. `启动系统.bat`

或使用 PowerShell：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe app.py
```

随后访问 http://127.0.0.1:8000。首次运行会自动创建 SQLite 数据库。

## 6. DeepSeek API 配置

复制 `.env.example` 为 `.env`，仅在部署机器本地填写配置：

```dotenv
ZONGCE_SECRET_KEY=请替换为足够长的随机字符串
ZONGCE_AI_ENABLED=1
DEEPSEEK_API_KEY=请填写有效的DeepSeek密钥
DEEPSEEK_API_URL=https://api.deepseek.com/chat/completions
DEEPSEEK_MODEL=deepseek-chat
```

修改后重新创建容器：

```powershell
docker compose up -d --force-recreate
```

注意：

- `.env`、真实密钥和 `.zongce_secret` 不得提交到公开仓库或放入交付压缩包。
- 密钥泄露后应立即在 DeepSeek 控制台作废并重新生成。
- AI 默认在 Docker 演示环境中关闭；配置密钥并设置 `ZONGCE_AI_ENABLED=1` 后启用。

## 7. API 使用说明

API 基础地址：

```text
http://127.0.0.1:8000
```

交互式接口文档：

```text
http://127.0.0.1:8000/docs
```

登录接口示例：

```http
POST /api/login
Content-Type: application/json

{
  "username": "admin",
  "password": "123456"
}
```

登录成功后，从响应中取得 `accessToken`。调用受保护接口时携带：

```http
Authorization: Bearer <accessToken>
```

主要接口分组：

| 分组 | 功能示例 |
|---|---|
| `/api/login`、`/api/user/*` | 登录和个人资料 |
| `/api/admin/student/*` | 单个新增、Excel 导入、查询和管理学生 |
| `/api/application/*` | 创建、修改、提交、撤回申请 |
| `/api/file/*` | 证明材料上传和鉴权访问 |
| `/api/audit/*` | 教师审核和审核记录 |
| `/api/rule/*` | 加分规则查询与维护 |
| `/api/accounting/*` | 学生/班级核算结果 |
| `/api/admin/accounting/*` | 扣分、终审、排名和 Excel 导出 |
| `/api/batch/*` | 学生 Excel 批量申报 |
| `/api/ai/*` | 单条/批次材料脱敏与 AI 建议 |
| `/api/announcement/*` | 成绩公示查询 |
| `/api/objection/*` | 学生异议提交与处理 |
| `/api/report/*` | 个人综合测评 PDF 报告 |

实际请求字段、响应结构和权限要求以 `/docs` 自动生成文档为准。统一响应格式为：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

## 8. 自动化测试

非 Docker 环境安装依赖后执行：

```powershell
.\.venv\Scripts\python.exe -m unittest discover -s tests -v
```

测试覆盖登录锁定、角色与班级权限、申请状态机、材料校验、教师审核、核算封顶、终审锁定、批量申报、材料脱敏、AI 不得改分、排名、公示异议、重复材料和 PDF 报告。

## 9. 推荐验收流程

1. 执行 `docker compose up --build -d`。
2. 管理员新增学生或检查班级、教师和规则。
3. 学生创建申请、上传材料并提交。
4. 学生发起材料脱敏与 AI 辅助核验（配置 API 后）。
5. 教师查看材料和 AI 建议，退回或审核通过。
6. 管理员核算、终审、查看排名并发布公示。
7. 学生查看结果、提交异议并下载个人 PDF。
8. 管理员导出 Excel 并展示重复材料检测。

## 10. 交付安全说明

本交付包不包含以下内容：

- `.env` 和 DeepSeek 密钥
- `.zongce_secret`
- 本机 `.venv`
- 真实 `zongce.db`
- `uploads`、真实学生证明材料和备份文件

Docker 演示数据使用虚构学号和脱敏姓名，可安全用于本地功能展示。正式部署前应配置强随机密钥、HTTPS、可靠数据库备份和访问日志管理。
